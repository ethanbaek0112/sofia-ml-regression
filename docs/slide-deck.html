TL;DR

  • Problem: 15 anonymized features → one real-valued target, scored by R². Very weak signal, extreme outliers.
  • Main idea: don't fight outliers with a fancier model — clean the training target (winsorize) and keep a simple Ridge.
  • Result: Public LB 0.04843, beat the baseline (0.04599); was #1 on public for a while.
  • Final twist: on the private split my rank didn't move at all (public 6th → private 6th) while the top reshuffled wildly.

EDA

  • 2500 train rows × 15 anonymized features (x0–x14), ~1.2% missing per column.
  • The target is brutal:
    • range −41,008 to +69,628, but most values sit in −50…+50
    • skewness 13.6, kurtosis 708
    • 0.4% of rows have |y| > 10,000
  • Weak signal: max |correlation| with the target is 0.25 (x9). Features barely correlate with each other either.

Target distribution — extreme tails

Takeaway: weak signal + strong outliers → a linear model gets wrecked by a handful of extreme rows.

Experiments & what failed (and why)

I started inside the lecture toolkit (Lec 02/03/06/07) and let the failures guide me.

  • Plain Linear Regression → CV R² = −0.46. One fold hit −2.41 because the outliers landed there — the "sensitive to outliers" weakness from Lec 07.
  • Polynomial + Ridge → −0.84. The polynomial terms just amplified the outliers.
  • Log-transformed target → −26. The exp() inverse blew small errors up — one test prediction reached −309,744.
  • Swept Ridge / LASSO / ElasticNet / k-NN → they all converged around +0.021.
Motivation that carried the project: when 3 different model families agree on the same ceiling, the limit is the data, not the model. So I stopped trying to be clever with the model.
Baek Seunghan

The fix: winsorize the target

  • Insight: outliers in the training data corrupt the fit, and I can't predict them in the test set anyway. So remove their influence from training only — never touch the test data.
  • Method: clip the training target to the [0.5%, 99.5%] percentiles, then fit Ridge (α=1).
  • Clip level = bias–variance tradeoff: too hard = lose information, too light = the outliers come back. [0.5, 99.5] was the sweet spot.
  • Evaluation note: I trusted out-of-fold (OOF) R², not cross_val_score, because one bad fold tanks the averaged version.

Reflection + code

What worked

  • Honest EDA → understood the outlier problem first
  • Multiple model families to find the true ceiling
  • Winsorizing — fix the data, not the model
  • OOF R² over cross_val_score

What didn't (all documented)

  • Polynomial features, log-target transform, heavy feature engineering, too-light winsor, trees/kernel methods

On the dataset (constructive)

  • R² ceiling ~0.05–0.08 → 90%+ of the variance is irreducible noise. Train/test same distribution (adversarial AUC ≈ 0.49) — not overfitting, the signal is just weak.
  • That's why the final ranking shuffled so much: rank was driven by variance as much as skill.
  • Honest note: I beat the baseline on public (0.0484 vs 0.0460) but landed just under it on private (0.0501 vs 0.0526). Most teams above the private baseline jumped from the bottom of the public board — variance, not a stable model. Mine was the only top entry that didn't move on either split.
  • A slightly stronger signal would let the board reward modeling over luck.
